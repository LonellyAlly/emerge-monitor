üìã Prerequisites

Required Dependencies

  • Gentoo Linux - The distribution this tool is designed for
  • bash - Version 5.0 or higher
  • genlop - For emerge progress tracking
  • sudo - Required to run genlop with proper permissions
# Install genlop
sudo emerge app-portage/genlop

Optional Dependencies (Screensavers)

Install any or all of these for the full screensaver experience:

# pipes.sh - Animated pipes
sudo emerge app-misc/pipes

# cmatrix - Matrix-style falling characters
sudo emerge app-misc/cmatrix

# hollywood - Hollywood-style hacker terminal
sudo emerge app-misc/hollywood

# asciiquarium - ASCII aquarium
sudo emerge games-misc/asciiquarium
Note: The script will work without these screensavers, but you'll only have the "none" option available. The script checks for available screensavers on startup.

üöÄ Installation

Step 1: Download the Script

wget https://raw.githubusercontent.com/yourusername/emerge-monitor/main/emerge-monitor.sh

Step 2: Make it Executable

chmod +x emerge-monitor.sh

Step 3: Optional - Install System-Wide

For easier access, move the script to your system path:

sudo mv emerge-monitor.sh /usr/local/bin/emerge-monitor
Done! You can now run the script using ./emerge-monitor.sh or emerge-monitor if installed system-wide.

üíª Usage

Basic Usage

Simply run the script while an emerge process is active:

./emerge-monitor.sh

Or if installed system-wide:

emerge-monitor

Running Without Emerge

You can run the script even without an active emerge process. It will display "No emerge process currently running" but all other features (screensavers, timers, controls) will work normally.

Typical Workflow

Terminal 1: Start your emerge

sudo emerge --update --deep --newuse @world

Terminal 2: Monitor the progress

./emerge-monitor.sh

‚å®Ô∏è Keyboard Controls

All controls are single-key presses. The script will respond immediately.

Key Action Description
s Toggle Screensaver Cycles through screensavers: none ‚Üí pipes.sh ‚Üí cmatrix ‚Üí hollywood ‚Üí asciiquarium ‚Üí none
r Manual Refresh Manually refreshes the emerge status (also exits screensaver mode)
a Toggle Auto-refresh Switches between automatic and manual refresh modes
+ or = Increase Interval Increases auto-refresh interval by 1 second
- Decrease Interval Decreases auto-refresh interval by 1 second (minimum: 1 second)
q Quit Exits the program cleanly

üìñ Detailed Feature Guide

1. Emerge ETA Monitoring

The main screen displays comprehensive information about your current emerge operation:

  • Current emerge progress: Shows which package is being compiled
  • ETA (Estimated Time of Arrival): Time remaining for current package
  • Overall progress: If multiple packages are being emerged
  • Compilation details: Build time estimates based on historical data
How it works: The script uses genlop -c to query the current emerge status. This requires sudo privileges because genlop needs to read portage logs in /var/log/emerge.log.

Example Output

Currently merging 1 out of 3

 * app-editors/vim-9.0.2116

       current merge time: 1 minute and 23 seconds.
       ETA: 2 minutes.

2. Screensaver Mode

Press s to cycle through different screensavers. This is perfect for keeping your terminal looking interesting while packages compile.

Available Screensavers:

1. none

Default view showing emerge status and information.

2. pipes.sh

Animated colorful pipes flowing across your terminal in various patterns.

3. cmatrix

Matrix-style cascading characters, just like the movie!

4. hollywood

Simulates a Hollywood hacker terminal with multiple split windows showing various "hacking" activities.

5. asciiquarium

A beautiful ASCII art aquarium with swimming fish, bubbles, and seaweed.

Pro Tip: Screensavers continue to run while emerge is working in the background. You won't miss any updates! Press r at any time to check the emerge status.

3. Real-time Clock & Timers

The header displays three important time-related metrics:

Current Time:     2025-10-30 14:32:15
Script Runtime:   00:15:32
Emerge Runtime:   00:42:18
  • Current Time: Your system's current date and time, updated every second
  • Script Runtime: How long the emerge-monitor script has been running (format: HH:MM:SS)
  • Emerge Runtime: How long the current emerge process has been running (detected automatically)

4. Auto-refresh vs Manual Refresh

Auto-refresh Mode (Default)

  • Screen updates automatically at the configured interval (default: 5 seconds)
  • Adjust interval using + and - keys
  • Shows "Auto (5 sec)" in the header
  • Best for: Continuous monitoring without interaction

Manual Refresh Mode

  • Screen only updates when you press r
  • Shows "Manual" in the header
  • Toggle with a key
  • Best for: Reading output carefully, slow terminals, or conserving resources

5. Configurable Refresh Interval

Fine-tune how often the screen updates in auto-refresh mode:

  • Press + or = to increase interval (5 ‚Üí 6 ‚Üí 7 ‚Üí ... seconds)
  • Press - to decrease interval (5 ‚Üí 4 ‚Üí 3 ‚Üí ... seconds)
  • Minimum interval: 1 second
  • Maximum interval: unlimited (though realistically you won't want more than 60 seconds)
Performance Tip: Increase the interval to reduce CPU usage on older systems or when running over SSH connections.

üîß Configuration

Sudo Configuration for Genlop

The script requires sudo access to run genlop. To avoid entering your password each time:

Step 1: Edit sudoers file

sudo visudo

Step 2: Add this line (replace yourusername)

yourusername ALL=(ALL) NOPASSWD: /usr/bin/genlop

Step 3: Save and exit

In vi/vim: Press ESC, then type :wq

Security Warning: Only do this if you understand the security implications. This allows running genlop without a password prompt.

Customizing the Script

You can edit the script to customize various aspects:

Default Refresh Interval

Change line 19 in the script:

REFRESH_INTERVAL=5  # Change to your preferred value

Color Scheme

Modify the color definitions at the top (lines 7-14):

RED='\033[0;31m'
GREEN='\033[0;32m'
YELLOW='\033[1;33m'
# ... etc

Screensaver Order

Rearrange the screensavers array (line 25):

SCREENSAVERS=("none" "cmatrix" "pipes.sh" "hollywood" "asciiquarium")

üêõ Troubleshooting

Problem: "genlop is not installed"

Solution: Install genlop using emerge:

sudo emerge app-portage/genlop

Problem: "No emerge process currently running"

This is normal if you run the script without an active emerge. The script will still work, but won't show emerge progress.

Solution: Start an emerge in another terminal:

sudo emerge --ask some-package

Problem: Screensaver not found

Solution: Install the missing screensaver (see Prerequisites) or simply skip to the next one by pressing s again.

# Example: Install cmatrix
sudo emerge app-misc/cmatrix

Problem: Screen doesn't refresh automatically

Possible causes and solutions:

  • Auto-refresh is disabled: Press a to enable it
  • You're in screensaver mode: Press r to return to main view
  • Interval is very long: Press - several times to reduce it

Problem: Terminal display issues or garbled text

Solutions:

  • Resize your terminal window (minimum recommended: 80 columns √ó 24 rows)
  • Press r to force a refresh
  • Exit and restart the script

Problem: Script exits immediately

Cause: Script must be run in an interactive terminal.

Solution: Don't run the script in a non-interactive shell or as a background process.

Problem: Permission denied errors

Cause: genlop requires sudo privileges to read emerge logs.

Solution: Configure sudoers as described in the Configuration section.
